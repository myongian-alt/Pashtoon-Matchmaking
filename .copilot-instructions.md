## Architecture Essentials

### Context System (Global State)
- **UserContext** (`src/context/UserContext.tsx`): Manages `selectedGender` ('male'|'female'|null)
- **FormContext** (`src/context/FormContext.tsx`): Manages 80+ profile fields with `updateField()` and `calculateProfileStrength()`
- **NotificationsContext**: Manages notifications state
- **Provider Order** (in AppNavigator): `FormProvider > UserProvider > NotificationsProvider > NavigationContainer`

### Navigation Type Safety
- All screens must be registered in `RootStackParamList` in [AppNavigator.tsx](src/navigation/AppNavigator.tsx)
- Pass params with type: `navigation.navigate('ScreenName', { param: value })`
- Import type: `import { RootStackParamList } from '../navigation/AppNavigator'`

### Screen Patterns

**Discovery Screens** (ProfileDiscoveryScreen)
- Show gender-based opposite profiles
- Read `selectedGender` from `useUser()` hook
- Filter: `selectedGender === 'male' ? femaleProfiles : maleProfiles`
- Include: Avatar badges, marital status, action buttons with labels

**Detail Screens** (ProfileDetailScreen)
- Display comprehensive profile with multiple sections
- Use section headers with icons (MaterialCommunityIcons)
- Sections: Personal, Physical, Education, Lifestyle, Values, Personality
- Gallery placeholder for photos

**Form Screens** (ProfileFormScreen)
- Multi-section tabbed interface
- Field types: `text`, `textarea`, `dropdown`, `yesno`, `avatar`, `gallery`
- Mandatory fields marked with red asterisks
- Profile strength percentage in header
- Call `calculateProfileStrength()` from FormContext

## Common Development Tasks

### Add a New Screen
1. Create file: `src/screens/NewScreen.tsx`
2. Add type to `RootStackParamList` in [AppNavigator.tsx](src/navigation/AppNavigator.tsx)
3. Import and add route in `AppNavigator`
4. Use `navigation.navigate('NewScreen', params)` to navigate

### Use FormContext in a Component
```tsx
import { useForm } from '../context/FormContext';

const MyComponent = () => {
  const { formData, updateField, calculateProfileStrength } = useForm();
  
  const strength = calculateProfileStrength();
  updateField('name', 'value');
  
  return <Text>{strength}%</Text>;
};
```

### Filter Profiles by Gender
```tsx
const { selectedGender } = useUser();
const profiles = selectedGender === 'male' ? femaleProfiles : maleProfiles;
```

### Add Back Button to Header
```tsx
<Pressable style={styles.backButton} onPress={() => navigation.goBack()}>
  <MaterialCommunityIcons name="chevron-left" size={28} color={theme.colors.primary} />
</Pressable>
```

## Troubleshooting

| Issue | Solution |
|-------|----------|
| Context not updating | Check Provider wrapping in AppNavigator; verify hook usage |
| Gender filtering broken | Verify `selectedGender` is set in ChooseGenderScreen; check `useUser()` hook |
| Navigation type errors | Add screen to `RootStackParamList` before navigating |
| Form field not saving | Verify field key matches `ProfileFormData` interface in FormContext |
| TypeScript errors | Run `npx tsc --noEmit` for detailed error locations |
